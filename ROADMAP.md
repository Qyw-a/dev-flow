# Branch Manager — 开发路线图

> 项目定位：面向多项目团队的 **需求-分支关联管理工具**，支持批量 Git 分支操作与工作流自定义。

---

## 一、当前已实现（v1.0.0）

### 1.1 项目管理
- [x] 添加/移除/列表项目目录
- [x] 自动检测是否为 Git 仓库
- [x] 项目配置持久化（`~/.branch-manager/projects.json`）

### 1.2 分支管理（Git 核心能力）
- [x] 查看本地分支列表（含当前分支高亮、最新提交信息）
- [x] **查看远程分支列表**（`origin/xxx` 前缀区分展示）
- [x] 创建分支（基于当前或指定分支）
- [x] 合并分支（支持 `--ff` / `--no-ff` 策略）
- [x] 推送到远程
- [x] 删除分支（支持强制删除 `-D`）
- [x] 切换分支（checkout）
- [x] 批量 Fetch / Pull

### 1.3 视图与交互
- [x] **按项目视图**：每个项目独立展示分支表格，支持行级多选
- [x] **按分支视图**：同名分支跨项目聚合，便于查看公共分支状态差异
- [x] 批量操作工具栏：批量创建、合并、推送、切换、删除
- [x] 操作日志面板：记录每次操作的成功/失败结果与时间

### 1.4 技术栈
- Electron 29 + React 18 + TypeScript 5
- Vite + electron-vite 构建
- Ant Design 5 + Zustand 状态管理
- simple-git（Node.js Git 封装）

## 二、近期规划（v1.1.x）

### 2.1 需求管理模块
> 核心目标：让"业务需求"成为管理分支的入口，而非直接操作 Git 分支。

- [ ] **需求（Ticket/Story）CRUD**
  - 创建需求卡片：标题、描述、优先级、状态（待开发 / 开发中 / 测试中 / 已上线）
  - 需求列表页，支持筛选与搜索
- [ ] **需求-分支关联**
  - 在需求详情页关联多个项目的具体分支
  - 一个需求可跨多个项目创建同名/异名分支
  - 分支表格展示所属需求标签
- [ ] **从需求一键创分支**
  - 选择需求 → 选择项目 → 输入分支名 → 批量在各项目中创建
  - 自动记录分支与需求的关联关系

### 2.2 分支能力增强
- [ ] **从远程分支检出**（`git checkout -b xxx origin/xxx`）
- [ ] **分支对比**：查看两个分支之间的 commit 差异
- [ ] **分支搜索/过滤**：在分支列表中按名称搜索
- [ ] **分支状态标记**：显示分支是否已推送、是否有本地未提交更改

### 2.3 数据持久化扩展
- [ ] 需求数据存储（`~/.branch-manager/tickets.json` 或 SQLite）
- [ ] 需求-分支关联关系存储

## 三、中期规划（v1.2.x）

### 3.1 工作流自定义配置
> 核心目标：不同团队/项目可以配置自己的分支规范与自动化规则。

- [ ] **工作流模板定义**
  - 配置分支类型与命名规范（如 `feature/{ticket-id}-描述`、`hotfix/描述`）
  - 配置分支生命周期规则（如 feature 必须从 develop 切出，必须合并回 develop）
  - 配置禁止直接操作的分支（如 `main`、`master` 保护规则）
- [ ] **项目级工作流绑定**
  - 每个项目可独立选择/绑定一个工作流模板
  - 创建分支时自动校验命名是否符合工作流规范
  - 合并操作自动校验目标分支是否符合工作流规则
- [ ] **工作流预设模板**
  - Git Flow（`main` / `develop` / `feature/*` / `release/*` / `hotfix/*`）
  - GitHub Flow（`main` + `feature/*`）
  - GitLab Flow（`main` / `production` + `feature/*`）
  - 支持用户自定义模板

### 3.2 操作增强与体验优化
- [ ] **标签/分组管理**：给项目打标签（如"前端组"、"后端组"），支持按标签筛选
- [ ] **快捷键支持**：常用操作绑定键盘快捷键
- [ ] **操作撤销/回滚**：关键操作（如批量删除）支持撤销
- [ ] **深色模式**：Ant Design 主题切换

## 四、长期规划（v2.0.x）

### 4.1 远程仓库集成
- [ ] **GitHub / GitLab / Gitee 集成**
  - OAuth 登录，拉取远程 PR/MR 列表
  - 本地分支与远程 PR 状态关联展示
  - 一键创建 PR / MR（基于当前分支）
- [ ] **CI/CD 状态展示**：在分支列表中显示最近一次构建/部署状态

### 4.2 多人协作
- [ ] 需求分配：将需求指派给具体开发者
- [ ] 分支锁定/抢占标记：标记某分支正在由谁开发
- [ ] 团队视图：查看团队成员当前所在分支及负责的需求

### 4.3 数据可视化
- [ ] 需求燃尽图 / 状态流转图
- [ ] 项目分支活跃度统计
- [ ] 工作流合规性检查报告
